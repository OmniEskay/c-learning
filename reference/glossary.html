C Programming Glossary

Every term used across this course, with definitions and code examples.

array
A contiguous block of memory holding a fixed number of elements of the same type. Accessed by zero-based index. The array name decays to a pointer to the first element in most contexts.
int scores[5] = {10, 20, 30, 40, 50};
printf("%d\n", scores[2]);  /* 30 */
→ Taught in: Arrays & Strings
buffer overflow
Writing data beyond the end of an allocated buffer, corrupting adjacent memory. One of the most common and dangerous security vulnerabilities in C programs.
char buf[5];
strcpy(buf, "Hello, World!");  /* writes 14 bytes into 5-byte buffer! */
→ Taught in: Arrays & Strings
calloc
Allocates memory for an array of n elements of the given size, zero-initialising all bytes. Returns NULL on failure. Part of <stdlib.h>.
int *arr = calloc(10, sizeof(int));  /* 40 bytes, all zeros */
if (!arr) { perror("calloc"); exit(1); }
free(arr);
→ Taught in: Memory Management
compiler
A program that translates C source code into machine code. GCC and Clang are the most common C compilers. The compilation process involves preprocessing, compilation, assembly, and linking.
gcc -Wall -o program main.c  # compile main.c into an executable
→ Taught in: Setup & Toolchain
dereference
Following a pointer to read or write the value at the address it holds. Done with the unary * operator. Dereferencing a NULL or invalid pointer causes a segfault.
int x = 42;
int *p = &x;
printf("%d\n", *p);  /* dereference: prints 42 */
*p = 100;            /* write through pointer */
→ Taught in: Pointers
EOF
End Of File. The value returned by fgetc() and similar functions when the file ends or an error occurs. Defined as -1 in <stdio.h>. Must be stored in an int, not a char, to distinguish from valid byte values.
int ch;
while ((ch = fgetc(fp)) != EOF) {
    putchar(ch);
}
→ Taught in: File I/O
errno
A global integer variable set by the C library when a system call or library function fails. Defined in <errno.h>. Only meaningful immediately after a failure. Thread-local in POSIX environments.
#include <errno.h>
#include <string.h>
FILE *fp = fopen("missing.txt", "r");
if (!fp) printf("Error: %s\n", strerror(errno));
→ Taught in: Error Handling
free
Releases heap memory previously allocated with malloc, calloc, or realloc. Calling free(NULL) is safe (does nothing). Double-free is a security vulnerability. After freeing, set the pointer to NULL.
int *p = malloc(sizeof(int));
*p = 42;
free(p);
p = NULL;  /* prevent use-after-free */
→ Taught in: Pointers
header file
A .h file containing declarations (function prototypes, struct definitions, macros, typedefs) shared across multiple .c files. Included with #include. Does not contain function bodies (those go in .c files).
#include <stdio.h>   /* standard library header */
#include "mymodule.h" /* your own header */
→ Taught in: Hello, World!
heap
A region of virtual memory used for dynamic allocation. Memory is allocated explicitly with malloc/calloc and released with free. Unlike the stack, heap memory persists across function calls until freed. Subject to fragmentation.
int *data = malloc(100 * sizeof(int));  /* allocate on heap */
/* ... use data ... */
free(data);  /* return to heap */
→ Taught in: Memory Management
linker
The tool that combines compiled object files (.o) and library archives (.a) into a final executable. Resolves symbol references between files. A "linker error" (undefined reference) means a function is declared but its definition isn't provided.
gcc main.o utils.o -o program  # linker combines two object files
→ Taught in: Setup & Toolchain
linked list
A data structure where each node holds a value and a pointer to the next node. Unlike arrays, nodes need not be contiguous. Supports O(1) head insertion/deletion but O(n) search.
typedef struct Node {
    int value;
    struct Node *next;
} Node;
→ Taught in: Dynamic Data Structures
malloc
Allocates n bytes of uninitialised heap memory. Returns a void* pointer that should be cast to the target type. Returns NULL if allocation fails. Must be paired with free() to avoid memory leaks.
int *arr = malloc(10 * sizeof(int));
if (!arr) { perror("malloc"); exit(1); }
/* ... use arr ... */
free(arr);
→ Taught in: Pointers
memory leak
Heap memory that was allocated but never freed. The memory is inaccessible to the program (the pointer was lost) but still reserved from the OS. Accumulates until the process is killed. Detected by Valgrind.
void leak(void) {
    int *p = malloc(100);
    /* forgot free(p) — 100 bytes leaked each call */
}
→ Taught in: Memory Management
null pointer
A pointer with value NULL (0). Does not point to any valid memory. Dereferencing a null pointer causes an immediate segfault. Used as a sentinel value to indicate "no pointer" or failure.
int *p = NULL;
if (p != NULL) *p = 5;  /* safe — checks before dereferencing */
→ Taught in: Pointers
null terminator
The '\0' byte (ASCII 0) that marks the end of a C string. Every C string must end with a null terminator. String functions like strlen, printf, and strcpy rely on it to know where the string ends.
char s[6] = "Hello";
/* s = {'H','e','l','l','o','\0'} */
printf("len=%zu\n", strlen(s));  /* 5 */
→ Taught in: Arrays & Strings
pointer
A variable that stores a memory address. Declared with * in the type (int *p). The & operator gives the address of a variable. The * operator dereferences a pointer to access the value at its address.
int x = 42;
int *p = &x;   /* p holds address of x */
printf("%d\n", *p);  /* 42 — value at the address */
→ Taught in: Pointers
preprocessor
The first phase of compilation. Processes directives starting with #: #include inserts file contents, #define creates macros, #ifdef/#ifndef enables conditional compilation. Runs before the C compiler sees your code.
#include <stdio.h>   /* preprocessor replaces this with stdio.h contents */
#define MAX 100      /* preprocessor replaces MAX with 100 everywhere */
→ Taught in: Setup & Toolchain
prototype
A forward declaration of a function's signature (return type and parameter types) without its body. Allows calling a function before its definition appears in the file. Usually placed in header files.
int square(int x);  /* prototype */
int main(void) { return square(5); }
int square(int x) { return x * x; }  /* definition */
→ Taught in: Functions
realloc
Resizes a previously allocated heap block. May move the data to a new address. Returns NULL on failure (original block is unchanged). Never assign realloc's result directly to the original pointer — use a temp variable.
int *arr = malloc(5 * sizeof(int));
int *tmp = realloc(arr, 10 * sizeof(int));  /* grow to 10 ints */
if (!tmp) { free(arr); exit(1); }
arr = tmp;  /* safe — tmp is valid */
→ Taught in: Memory Management
recursion
A function that calls itself. Requires a base case to terminate. Each call pushes a new stack frame. Deep recursion can overflow the stack. Used in tree traversal, divide-and-conquer algorithms, and parsing.
int factorial(int n) {
    if (n <= 1) return 1;           /* base case */
    return n * factorial(n - 1);    /* recursive case */
}
→ Taught in: Recursion
scope
The region of source code where a name (variable, function) is visible. Local variables have block scope (visible within their {} block). Global variables have file scope. Parameters have function scope.
int global = 1;        /* file scope */
void f(void) {
    int local = 2;     /* block scope — invisible outside f */
    {
        int inner = 3; /* block scope — invisible outside {} */
    }
}
→ Taught in: Functions
segfault (segmentation fault)
SIGSEGV — a crash caused by accessing memory the process does not own. Caused by: NULL dereference, out-of-bounds array access, use-after-free, stack overflow. Debug with GDB's backtrace command.
int *p = NULL;
*p = 5;  /* SIGSEGV — kills the process immediately */
→ Taught in: Memory Management
stack
A region of virtual memory for local variables and function call frames. Grows downward. Automatically managed — pushed on function entry, popped on return. Typically limited to ~8 MB. Deep recursion exhausts the stack.
void f(void) {
    int local = 42;  /* allocated on the stack */
}  /* local destroyed here — stack frame popped */
→ Taught in: Memory Management
struct
A user-defined type that groups related variables (fields) under one name. Fields are accessed with the . operator (struct variable) or -> operator (struct pointer). Passed by value by default — pass a pointer for efficiency.
typedef struct { char name[50]; int age; double gpa; } Student;
Student s = {"Alice", 20, 3.8};
printf("%s\n", s.name);
→ Taught in: Structures
typedef
Creates a type alias. Commonly used with structs to avoid writing "struct" everywhere. Also used for function pointer types and platform-independent integer types.
typedef struct { double x; double y; } Point;
Point p = {1.0, 2.0};  /* no "struct" keyword needed */
→ Taught in: Structures
undefined behaviour
Code that the C standard does not define — the compiler may do anything. Including: crash, produce wrong results, or appear to work correctly. Common causes: signed integer overflow, null dereference, out-of-bounds access, use-after-free. The most dangerous category of C bugs.
int arr[3] = {1, 2, 3};
int x = arr[10];  /* undefined behaviour — reads past array end */
int n = INT_MAX + 1;  /* undefined behaviour — signed overflow */
→ Taught in: Syntax & Data Types
