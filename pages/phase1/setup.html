Setup & Toolchain

Why does this matter?

The compiler is the single most important tool in a C programmer's life. Understanding GCC's flags and error messages transforms cryptic red text into actionable debugging information. Professional C developers use -Wall -Wextra -Werror to catch bugs at compile time before they become runtime crashes in production.

Installing the C Toolchain

Ubuntu / Debian:

sudo apt update
sudo apt install build-essential gdb valgrind
gcc --version

Fedora / RHEL:

sudo dnf groupinstall "Development Tools"
sudo dnf install valgrind gdb
gcc --version

Arch Linux:

sudo pacman -S base-devel gdb valgrind
gcc --version

Install Xcode Command Line Tools (includes Clang which is GCC-compatible):

xcode-select --install
clang --version   # or: gcc --version (symlinked to clang)

Install Homebrew for additional tools:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
brew install gcc gdb valgrind

Two options:

Option 1: WSL2 (recommended) — install Windows Subsystem for Linux, then follow the Linux instructions above.

wsl --install   # Run in PowerShell as Administrator

Option 2: MinGW-w64

  1. Download and install MSYS2
  2. In the MSYS2 terminal: pacman -S mingw-w64-x86_64-gcc
  3. Add C:\msys64\mingw64\bin to your PATH

Essential GCC Flags

FlagWhat it doesWhen to use
-o nameName the output binaryAlways
-WallEnable all common warningsAlways
-WextraEnable extra warnings beyond -WallAlways
-WerrorTreat warnings as errorsProduction code
-gInclude debug symbols for GDBDebugging
-O2Optimise for speedRelease builds
-std=c11Use C11 standardModern code
-fsanitize=addressAddressSanitizer — detect memory errorsTesting

A good default command for learning:

gcc -Wall -Wextra -g -std=c11 main.c -o main

VS Code Setup

Install these extensions:

  • C/C++ (Microsoft) — IntelliSense, debugging
  • C/C++ Extension Pack — includes cmake tools
  • clangd (optional) — faster, more accurate IntelliSense

Create .vscode/tasks.json for one-key builds:

{
  "version": "2.0.0",
  "tasks": [{
    "label": "build",
    "type": "shell",
    "command": "gcc -Wall -Wextra -g -std=c11 ${file} -o ${fileDirname}/${fileBasenameNoExtension}",
    "group": { "kind": "build", "isDefault": true }
  }]
}

Press Ctrl+Shift+B to build, F5 to debug.

Basic Program Structure

/* Every C program needs at minimum these elements */
#include <stdio.h>   /* Include standard I/O library declarations */

int main(void) {     /* Entry point — the OS calls main() to start */
    printf("Hello!\n");
    return 0;        /* 0 = success; non-zero = error code */
}

Every C program must have a main() function. The #include directive tells the preprocessor to copy the contents of stdio.h into your file, giving you access to printf and friends.

Diff: Missing #include → Compiler Error

/* BROKEN: missing #include */

int main(void) {
    printf("Hello, World!\n");
    return 0;
}
/* FIXED: include stdio.h */
#include <stdio.h>

int main(void) {
    printf("Hello, World!\n");
    return 0;
}
What was wrong: Without #include <stdio.h>, the compiler does not know what printf is. This produces an "implicit declaration of function 'printf'" warning or error.

Why the fix works: stdio.h contains the prototype int printf(const char *format, ...);. Including it tells the compiler the exact signature of printf so it can type-check your call and link to the correct library function.
⚠ Common Mistakes
  • Ignoring warnings — GCC warnings are almost always indicating real bugs. Enable -Wall and fix every warning before moving on.
  • Missing semicolons — C statements end with ;. Forgetting one causes confusing errors on the next line, not the line with the bug.
  • Compiling the wrong file — If you edit main.c but compile old.c, the binary won't reflect your changes. Always check the filename in your compile command.

Practice Exercises

Basic Compile with different flags

Write a minimal C program with a deliberate warning (e.g., declare an unused variable int x;). Compile it without -Wall, then with -Wall. Notice the difference.

Show hint
int x; inside main creates an "unused variable" warning with -Wall. Try also -Werror to see it become a compile error.
Intermediate Explore the compilation stages

Using a simple C file, run each stage separately and inspect the output:
gcc -E main.c -o main.i (preprocessed)
gcc -S main.c -o main.s (assembly)
gcc -c main.c -o main.o (object file)
Open each output file and describe what you see.

Show hint
main.i is a large C file with all includes expanded. main.s is assembly (x86 or ARM instructions). main.o is binary — use objdump -d main.o to disassemble it.
Challenge Write a Makefile

Create a Makefile that compiles a C project with multiple source files. It should have targets: all, clean, and correctly handle recompilation when only one file changes.

Show hint
Use pattern rules: %.o: %.c. The linker step combines all .o files. The clean target runs rm -f *.o program. Use automatic variables $@ (target) and $< (first prerequisite).

Checkpoint Quiz

Test your understanding

1. Which flag enables all common warnings in GCC?
-Wall enables a broad set of warnings. Note: -w (lowercase) actually suppresses warnings — don't confuse them!
2. What does the -o flag do in gcc main.c -o program?
-o specifies the output file name. Without it, GCC defaults to creating a.out (or a.exe on Windows).
3. What stage converts .c source to a .o object file?
Compilation (-c flag) translates C to machine-code object files. Linking then combines object files with library code into an executable.
