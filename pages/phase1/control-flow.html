Control Flow

Why does this matter?

Every algorithm you will ever write — sorting, searching, parsing, networking — is built from control flow: making decisions (if/else), repeating actions (loops), and dispatching on values (switch). The off-by-one error in a loop is one of the most common bugs in systems programming. Getting loops exactly right is the difference between correct code and data corruption.

if / else

#include <stdio.h>

int main(void) {
    int score = 75;

    if (score >= 90) {
        printf("Grade: A\n");
    } else if (score >= 80) {
        printf("Grade: B\n");
    } else if (score >= 70) {
        printf("Grade: C\n");
    } else {
        printf("Grade: F\n");
    }

    return 0;
}
if/else flow: score = 75 │ ┌─────▼─────┐ │ score≥90? │ NO └─────┬─────┘ │ ┌─────▼─────┐ │ score≥80? │ NO └─────┬─────┘ │ ┌─────▼─────┐ │ score≥70? │ YES → print "Grade: C" └─────┬─────┘ │ (else branch skipped)

switch

Use switch when dispatching on an integer or character value with many possible exact matches. It compiles to an efficient jump table, unlike a long chain of if/else if.

#include <stdio.h>

int main(void) {
    char op = '+';
    int a = 10, b = 3;

    switch (op) {
        case '+':
            printf("%d\n", a + b);
            break;          /* CRITICAL: without break, falls through to next case */
        case '-':
            printf("%d\n", a - b);
            break;
        case '*':
            printf("%d\n", a * b);
            break;
        case '/':
            if (b != 0) printf("%d\n", a / b);
            break;
        default:
            printf("Unknown operator\n");
            break;
    }

    return 0;
}
⚠ Fall-Through: Intentional vs Accidental

Without break, execution falls through to the next case's code. This is occasionally intentional (to share code between cases) but usually a bug. Always add break at the end of each case unless you explicitly intend fall-through, and add a comment /* fall through */ to make it clear.

for Loop

Use for when you know the exact iteration count ahead of time.

#include <stdio.h>

int main(void) {
    /* Count from 0 to 9 */
    for (int i = 0; i < 10; i++) {
        printf("%d ", i);
    }
    printf("\n");

    /* Iterate backwards */
    for (int i = 9; i >= 0; i--) {
        printf("%d ", i);
    }
    printf("\n");

    /* Sum 1 to 100 */
    int sum = 0;
    for (int i = 1; i <= 100; i++) {
        sum += i;
    }
    printf("Sum 1-100 = %d\n", sum);  /* 5050 */

    return 0;
}
for (init; condition; update) init: i = 0 │ ┌────▼──────────┐ │ condition: i<10│ ─── false ──→ exit loop └────┬──────────┘ │ true ┌────▼──────┐ │ body │ └────┬──────┘ │ ┌────▼──────┐ │ update i++│ └────┬──────┘ │ └──────────────→ check condition again

while Loop

Use while when the loop count is not known in advance — for example, reading until end-of-file or until a condition is met.

#include <stdio.h>

int main(void) {
    int n = 1;
    while (n <= 5) {
        printf("%d ", n);
        n++;
    }
    printf("\n");

    /* Read integers until user enters 0 */
    int input;
    int total = 0;
    printf("Enter numbers (0 to stop):\n");
    scanf("%d", &input);
    while (input != 0) {
        total += input;
        scanf("%d", &input);
    }
    printf("Total: %d\n", total);
    return 0;
}

do-while Loop

Like while, but the condition is checked after the body executes. The body always runs at least once. Useful for menus where you want to show options before asking the user to choose.

#include <stdio.h>

int main(void) {
    int choice;
    do {
        printf("1. Start\n2. Help\n3. Quit\nChoice: ");
        scanf("%d", &choice);
        if (choice == 1) printf("Starting...\n");
        else if (choice == 2) printf("Help text...\n");
    } while (choice != 3);

    printf("Goodbye!\n");
    return 0;
}

break and continue

#include <stdio.h>

int main(void) {
    /* break: exit the loop immediately */
    for (int i = 0; i < 10; i++) {
        if (i == 5) break;        /* stop at 5 */
        printf("%d ", i);         /* prints: 0 1 2 3 4 */
    }
    printf("\n");

    /* continue: skip the rest of this iteration */
    for (int i = 0; i < 10; i++) {
        if (i % 2 == 0) continue; /* skip even numbers */
        printf("%d ", i);          /* prints: 1 3 5 7 9 */
    }
    printf("\n");

    return 0;
}

Diff: Off-by-One Loop Error

/* BROKEN: off-by-one, misses last element */
#include <stdio.h>
#define N 5

int main(void) {
    int arr[N] = {10, 20, 30, 40, 50};
    int sum = 0;
    for (int i = 0; i < N - 1; i++) {  /* BUG: should be i < N */
        sum += arr[i];
    }
    printf("Sum: %d\n", sum);  /* prints 100, not 150 */
    return 0;
}
/* FIXED: correct upper bound */
#include <stdio.h>
#define N 5

int main(void) {
    int arr[N] = {10, 20, 30, 40, 50};
    int sum = 0;
    for (int i = 0; i < N; i++) {     /* FIXED: i < N */
        sum += arr[i];
    }
    printf("Sum: %d\n", sum);  /* prints 150 */
    return 0;
}
What was wrong: i < N - 1 stops at index 3 (0, 1, 2, 3), skipping index 4 (the last element, value 50). This is the classic off-by-one error — the loop exits one iteration too early.

Why the fix works: i < N runs for indices 0 through N-1, which is all N elements. The rule of thumb for C arrays: always use i < N, not i <= N-1 (they're equivalent but the former is clearer) and never i <= N (that's a buffer overflow).
⚠ Common Mistakes
  • Off-by-one errors — The most common loop bug. Always mentally check: does the loop run for the first element? The last element? One too many?
  • Infinite loops — Forgetting to update the loop variable (while (n < 10) without n++) creates an infinite loop. Always ensure the condition can become false.
  • Missing break in switch — Without break, execution falls through all remaining cases. This is almost always a bug.

Practice Exercises

Basic FizzBuzz

Print numbers 1 to 100. For multiples of 3, print "Fizz"; for multiples of 5, print "Buzz"; for multiples of both, print "FizzBuzz".

Show hint
Check divisibility with the modulo operator: n % 3 == 0. Check the combined case first (n % 15 == 0 or check both conditions with &&).
Intermediate Multiplication table

Use nested for loops to print a 10×10 multiplication table, with numbers aligned in columns using printf width specifiers.

Show hint
Use printf("%4d", i * j) for 4-character-wide columns. The outer loop is rows (1-10), the inner loop is columns (1-10). Print a newline after each row.
Challenge Collatz sequence

Implement the Collatz conjecture: start with any positive integer n. If n is even, divide by 2. If n is odd, multiply by 3 and add 1. Repeat until n = 1. Count and print the number of steps. Find which starting number under 1000 produces the longest sequence.

Show hint
Use a while loop for the sequence. Wrap it in a for loop from 1 to 999. Track the max steps seen and the starting number that produced them.

Checkpoint Quiz

Test your understanding

1. Which loop always executes its body at least once?
do-while checks its condition after the body, so the body always runs at least once — even if the condition is immediately false.
2. What keyword exits a loop immediately?
break exits the innermost loop or switch. continue skips to the next iteration. return exits the entire function. exit() terminates the whole program.
3. A switch statement requires what at the end of each case to prevent fall-through?
Without break, execution falls through to the next case's code. break transfers control to after the switch statement.
