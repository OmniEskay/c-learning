Introduction to C

Why does this matter?

C powers the Linux kernel — the foundation of every Android phone, web server, and supercomputer on the planet. When you type a command in a terminal, open a file, or run any program, the operating system handling that request is mostly written in C. Learning C means learning the language at the foundation of modern computing.

What is C?

C is a general-purpose compiled programming language created in 1972 by Dennis Ritchie at Bell Labs. It was originally written to develop the Unix operating system — and that lineage defines its character: C is designed to be close to the hardware, fast, and portable across different machines.

C occupies a unique position. It is not as low-level as assembly (you don't write individual CPU instructions), but not as abstract as Python or Java (there is no runtime managing memory for you). This makes C a "mid-level" language — expressive enough to write complex programs, yet close enough to the metal to control exactly what the CPU and memory do.

A Brief History

YearMilestone
1972Dennis Ritchie creates C at Bell Labs to rewrite Unix
1978"The C Programming Language" by Kernighan & Ritchie — the K&R book
1989ANSI C (C89) — first official standard, enabling portability
1999C99 — added inline functions, variable-length arrays, // comments
2011C11 — threading support, generic macros, bounds-checking
2023C23 — latest standard, improving type safety and modern syntax

Where is C Used Today?

C is not a "legacy" language gathering dust. It is actively used in:

  • Operating systems — Linux kernel, macOS/iOS kernel (XNU), Windows NT kernel
  • Embedded systems — Arduino, STM32 microcontrollers, automotive ECUs
  • Databases — SQLite, PostgreSQL, MySQL core engines
  • Language runtimes — CPython (Python interpreter), Ruby MRI, LuaJIT
  • Networking — OpenSSL, nginx, Redis, memcached
  • Game engines — portions of Quake, Doom, id Tech engines
⚠ Common Misconception

C is not just a "learning language." It is production software infrastructure used by billions of devices. The skills you develop here are directly applicable to systems programming careers.

The C Compilation Model

Unlike Python (which runs code line by line), C is compiled. Your source code goes through several stages before it becomes a runnable program:

Preprocessing → Compilation → Assembly → Linking ───────────────────────────────────────────────────── hello.c hello.i hello.s hello.o (source) (preprocessed) (assembly) (object file) │ ┌──────▼──────┐ libc │ Linker │ → hello (binary) ──────┘ │ └─ runnable executable
  • Preprocessor: Expands #include directives and #define macros
  • Compiler: Translates C into assembly language
  • Assembler: Converts assembly to machine-code object files
  • Linker: Combines object files with library code into a final executable

In practice, gcc hello.c -o hello does all four steps automatically. You only need to think about the stages when diagnosing errors — a "linker error" is different from a "compiler error."

Why Learn C in 2024+?

Several compelling reasons:

  1. Mental model of computing — C forces you to understand memory, pointers, and data representation. These concepts apply in every language you'll ever use.
  2. Performance-critical code — When nanoseconds matter (trading systems, game engines, signal processing), C is often the answer.
  3. Understanding your tools — Most developer tools (editors, shells, compilers) are written in C. Reading and contributing to these becomes possible.
  4. Embedded/IoT development — C is the only viable choice for many microcontrollers.
  5. Foundation for C++/Rust — Both languages inherit C's syntax and mental models. C is the fastest path to understanding them deeply.

By the end of the four phases, you will have built:

Practice Exercises

Basic Identify the compilation stage

For each of the following GCC flags, identify which compilation stage it stops at:
gcc -E, gcc -S, gcc -c, gcc (no flag)

Show hint
-E = preprocessor only; -S = compile to assembly; -c = compile to object file; no flag = full build including linking.
Intermediate Research two C projects

Pick two projects from the list (Linux, SQLite, Redis, CPython). Look up their GitHub repositories and identify: how large the codebase is (lines of code), how old it is, and one interesting design decision documented in their README or contributing guide.

Show hint
SQLite is a great choice — its documentation is excellent and it has a fascinating "lemon" parser and WAL journal design. Redis is also well-documented.
Challenge Trace a system call

On Linux, run strace echo "hello". Find the write system call in the output. Research what this system call does at the kernel level and which C function in stdio.h ultimately calls it when you use printf.

Show hint
printf → fwrite → write (syscall). The write() syscall takes a file descriptor, a buffer address, and a byte count. File descriptor 1 is stdout.

Checkpoint Quiz

Test your understanding

1. What year was C first standardized by ANSI?
ANSI C (C89) was standardized in 1989, giving portability across compilers. The K&R book came in 1978 but that predates the standard.
2. Which of the following is NOT a direct use of C?
Java bytecode is compiled by the JVM, which is itself written in C/C++, but the bytecode compilation is a Java concept, not C code you'd write directly.
3. C is classified as a:
C is often called mid-level — it has high-level constructs (functions, loops) but allows direct memory manipulation like a low-level language.
