Hello, World!

Why does this matter?

Hello World is a sanity check for your entire toolchain — compiler, linker, runtime, and terminal. Every time you set up C on a new machine, building and running Hello World confirms everything is wired up correctly. In embedded systems, the equivalent is blinking an LED — it proves your code is actually executing on the target hardware.

The Complete Hello World Program

#include <stdio.h>

int main(void) {
    printf("Hello, World!\n");
    return 0;
}

Save this as hello.c. Now compile and run it:

gcc -Wall -o hello hello.c
./hello
Hello, World!

Line-by-Line Walkthrough

#include <stdio.h>

This is a preprocessor directive. Before the compiler sees your code, the preprocessor replaces #include <stdio.h> with the entire contents of the header file stdio.h. That header contains the declaration (prototype) for printf:

int printf(const char *restrict format, ...);

Without this, the compiler doesn't know printf exists and will either warn or error out.

int main(void)

This declares the main function. Every C program must have exactly one main — it is the entry point, the function the operating system calls when it starts your program.

  • int — the return type. main() returns an integer to the OS.
  • main — the name (special — the OS looks for this).
  • void — this version takes no command-line arguments. Alternatively: int main(int argc, char *argv[]).

printf("Hello, World!\n");

printf is a function call. It is declared in stdio.h and defined in the C standard library (libc). The argument "Hello, World!\n" is a string literal — an array of characters stored in the program's read-only data segment.

The \n is an escape sequence representing a newline character (ASCII value 10). Without it, the next shell prompt would appear on the same line as your output.

return 0;

This returns the integer 0 from main() to the operating system. By convention: 0 = success, non-zero = an error code. The shell can check this: echo $? prints the last program's exit code.

How printf Works Under the Hood

When you call printf("Hello, World!\n"):

  1. The CPU jumps to the printf function in libc
  2. printf formats the string
  3. It calls the write system call: write(1, "Hello, World!\n", 14)
  4. The OS kernel copies those 14 bytes to the terminal (file descriptor 1 = stdout)
  5. The terminal displays them

Diff: Wrong Return Type on main()

/* BROKEN: void return type */
#include <stdio.h>

void main() {
    printf("Hello!\n");
    /* no return */
}
/* FIXED: int return type */
#include <stdio.h>

int main(void) {
    printf("Hello!\n");
    return 0;
}
What was wrong: void main() is non-standard. The C standard requires main() to return int. Many compilers will warn or error on void main(), and the exit code returned to the shell is undefined behaviour.

Why the fix works: int main(void) explicitly declares the return type as int and return 0 communicates success to the OS. The void in the parameter list is not required but makes it explicit that no arguments are accepted.
⚠ Common Mistakes
  • Forgetting \n — Without it, stdout may not flush immediately, especially when piping output. Always end printf output with \n or call fflush(stdout).
  • printf vs puts — For simple strings without formatting, puts("Hello\n") adds a newline automatically. But puts always adds a newline, so don't include one in the string too.
  • Missing semicolons on return — return 0 without a semicolon is a syntax error. return 0; is correct.

Practice Exercises

Basic Print multiple lines

Modify Hello World to print three lines: your name, today's date, and "Learning C!". Each on its own line.

Show hint
Call printf three times, or use a single printf with three \n characters. Try both approaches.
Intermediate Check the exit code

Change return 0; to return 42;. Compile and run, then immediately run echo $? in the terminal. What do you see? Now change it back to 0.

Show hint
$? holds the exit code of the last command. You should see 42. Shell scripts use this to check if a program succeeded.
Challenge Print a formatted table

Use printf with width specifiers to print a neat table of powers of 2 from 2¹ to 2¹⁰. Each row should show the power and the value, aligned in columns.

Show hint
Use printf("%3d %6d\n", power, value) for right-aligned columns. Width and precision can be specified: %8.2f means 8 chars wide, 2 decimal places.

Checkpoint Quiz

Test your understanding

1. What does return 0; at the end of main() signal?
By convention, returning 0 from main() tells the OS the program exited successfully. Non-zero values signal various error conditions.
2. What header must you include to use printf?
printf is declared in <stdio.h> (standard input/output). stdlib.h has malloc/free, string.h has string functions.
3. What does \n inside a string literal do?
\n is the escape sequence for a newline (ASCII 10). \t is tab. \0 is the null terminator. \\ is a literal backslash.
