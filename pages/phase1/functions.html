Functions

Why does this matter?

Every operating system call you make — open(), read(), malloc() — is a function. Understanding how functions work at the stack level is prerequisite knowledge for debugging with GDB, reading crash dumps, understanding recursion, and grasping why C passes everything by value. Function stack frames are the foundation of the call stack you see in every debugger.

Defining and Calling Functions

#include <stdio.h>

/* Function definition */
int add(int a, int b) {
    return a + b;
}

double average(int arr[], int len) {
    int sum = 0;
    for (int i = 0; i < len; i++) {
        sum += arr[i];
    }
    return (double)sum / len;
}

int main(void) {
    int result = add(3, 7);
    printf("3 + 7 = %d\n", result);

    int numbers[] = {10, 20, 30, 40, 50};
    printf("Average: %.1f\n", average(numbers, 5));

    return 0;
}

Function Prototypes

C reads files top-to-bottom. If you call a function before it's defined, the compiler doesn't know its signature. A prototype solves this by declaring the function signature early:

#include <stdio.h>

/* Prototypes — declare signature before main */
int multiply(int a, int b);
void print_result(int n);

int main(void) {
    int r = multiply(4, 5);  /* compiler knows multiply() from prototype */
    print_result(r);
    return 0;
}

/* Definitions can come after main */
int multiply(int a, int b) {
    return a * b;
}

void print_result(int n) {
    printf("Result: %d\n", n);
}

In real projects, prototypes live in header files (.h). The #include directive pulls them in wherever needed.

Pass by Value

C passes all arguments by value — the function receives a copy. Modifying the parameter does not affect the original variable:

#include <stdio.h>

void try_to_double(int x) {
    x = x * 2;         /* only modifies the local copy */
    printf("Inside: %d\n", x);
}

int main(void) {
    int n = 10;
    try_to_double(n);
    printf("Outside: %d\n", n);  /* still 10! */
    return 0;
}

To actually modify a variable from a function, you pass a pointer to it (covered in the Pointers lesson).

Scope

Scope determines where a variable is visible:

#include <stdio.h>

int global_count = 0;  /* global scope — visible everywhere */

void increment(void) {
    global_count++;     /* can access global */
    int local = 5;      /* local scope — only visible inside increment() */
}

int main(void) {
    int x = 10;         /* local to main */
    {
        int y = 20;     /* local to this inner block */
        printf("x=%d y=%d\n", x, y);
    }
    /* y is NOT visible here */
    printf("x=%d global=%d\n", x, global_count);
    return 0;
}

Recursive Functions

A function can call itself — this is recursion. Every recursive function needs a base case to terminate:

#include <stdio.h>

/* Compute n! (factorial) */
int factorial(int n) {
    if (n <= 1) return 1;        /* base case */
    return n * factorial(n - 1); /* recursive case */
}

int main(void) {
    for (int i = 0; i <= 6; i++) {
        printf("%d! = %d\n", i, factorial(i));
    }
    return 0;
}

Diff: Missing Function Prototype

/* BROKEN: function used before defined */
#include <stdio.h>

int main(void) {
    /* Compiler doesn't know square() yet! */
    int s = square(5);
    printf("%d\n", s);
    return 0;
}

int square(int x) {
    return x * x;
}
/* FIXED: prototype before main */
#include <stdio.h>

int square(int x);   /* prototype */

int main(void) {
    int s = square(5);
    printf("%d\n", s);
    return 0;
}

int square(int x) {
    return x * x;
}
What was wrong: In C89, calling a function before its definition or prototype was an "implicit declaration" — the compiler assumed it returned int and guessed the argument types. In C99+ this is an error. Even in C89, the implicit declaration is wrong if the function returns a non-int type.

Why the fix works: The prototype int square(int x); tells the compiler the exact return type and parameter type before main() calls it. The compiler can then verify your call site is correct.
⚠ Common Mistakes
  • Expecting pass-by-reference — C copies arguments. If you want to modify a variable in the caller, pass a pointer to it (void swap(int *a, int *b)).
  • Returning a pointer to a local variable — Local variables are destroyed when the function returns. Returning a pointer to one gives you a dangling pointer pointing to freed stack memory.
  • Missing return in non-void function — If a non-void function reaches the end without a return statement, the return value is undefined. GCC warns with -Wall: "control reaches end of non-void function".

Practice Exercises

Basic Min and Max functions

Write two functions: int min(int a, int b) and int max(int a, int b). Then write a third function int clamp(int val, int lo, int hi) that uses min and max to constrain a value to a range.

Show hint
clamp(val, lo, hi) = min(max(val, lo), hi). Alternatively, use if/else directly.
Intermediate Prime checker

Write a function int is_prime(int n) that returns 1 if n is prime and 0 otherwise. Use it to print all prime numbers up to 100.

Show hint
A number n is prime if it has no divisors other than 1 and n. Only check up to sqrt(n) — if n has a divisor larger than sqrt(n), it also has one smaller. Use: #include <math.h> and link with -lm.
Challenge Implement your own power function

Implement double my_pow(double base, int exp) both iteratively and recursively. Handle negative exponents. Compare your results against pow() from <math.h>.

Show hint
For negative exponents: my_pow(x, -n) = 1.0 / my_pow(x, n). Recursive: base case is exp==0 returns 1.0; recursive case returns base * my_pow(base, exp-1). Iterative: multiply base by itself exp times using a loop.

Checkpoint Quiz

Test your understanding

1. What is a function prototype?
A prototype declares the return type and parameter types so the compiler can verify calls before seeing the definition. It ends with a semicolon.
2. Variables declared inside a function have what scope?
Local variables live on the stack and are destroyed when their enclosing block exits.
3. What does void as a return type indicate?
void means no return value. A void function can still use a bare return; to exit early.
