Syntax & Data Types

Why does this matter?

C's type system is not just syntax — it maps directly to hardware. An int is 4 bytes on most platforms; a char is always 1 byte. Understanding types means understanding memory layout, which matters when you're reading data from a file, a network packet, or a hardware register. A mismatch between what you declared and what you stored is one of the most common sources of security vulnerabilities (buffer overflows, type confusion bugs).

Variables and Declarations

In C, every variable must be declared with a type before use. The type tells the compiler how many bytes to allocate and how to interpret the bit pattern stored there.

#include <stdio.h>

int main(void) {
    int age = 25;           /* 4 bytes, signed integer */
    float temperature = 98.6f; /* 4 bytes, floating point */
    double pi = 3.14159265; /* 8 bytes, double precision */
    char grade = 'A';       /* 1 byte, character */
    char name[] = "Alice";  /* array of chars (string) */

    printf("Name: %s, Age: %d\n", name, age);
    printf("Temp: %.1f, Grade: %c\n", temperature, grade);
    return 0;
}

Constants

Two ways to define constants in C:

/* Method 1: const keyword — type-checked, preferred */
const int MAX_SIZE = 100;
const double GRAVITY = 9.81;

/* Method 2: #define macro — textual substitution */
#define BUFFER_SIZE 1024
#define PI 3.14159

/* Using them */
char buffer[BUFFER_SIZE];  /* 1024-byte array */
double area = PI * radius * radius;

Prefer const over #define for typed constants — the compiler can type-check them and they appear in debuggers with their original names.

Operators

Arithmetic

int a = 17, b = 5;
printf("%d\n", a + b);   /* 22 — addition */
printf("%d\n", a - b);   /* 12 — subtraction */
printf("%d\n", a * b);   /* 85 — multiplication */
printf("%d\n", a / b);   /* 3  — integer division (truncates!) */
printf("%d\n", a % b);   /* 2  — modulo (remainder) */
⚠ Integer Division Trap

17 / 5 in C is 3, not 3.4. When both operands are integers, the result is an integer (truncated toward zero). To get float division: (double)17 / 5 or 17.0 / 5.

Comparison and Logical

int x = 10, y = 20;

/* Comparison — return 1 (true) or 0 (false) */
x == y   /* 0 — not equal */
x != y   /* 1 — not equal */
x < y    /* 1 — less than */
x >= y   /* 0 — greater or equal */

/* Logical */
(x < 15) && (y > 15)  /* 1 — AND: both true */
(x > 15) || (y > 15)  /* 1 — OR: at least one true */
!(x == y)              /* 1 — NOT */

Assignment and Shorthand

int n = 10;
n += 5;   /* n = n + 5 = 15 */
n -= 3;   /* n = n - 3 = 12 */
n *= 2;   /* n = n * 2 = 24 */
n /= 4;   /* n = n / 4 = 6  */
n %= 4;   /* n = n % 4 = 2  */
n++;      /* n = n + 1 = 3  (post-increment) */
++n;      /* n = n + 1 = 4  (pre-increment)  */
n--;      /* n = n - 1 = 3  */

Reading Input with scanf

#include <stdio.h>

int main(void) {
    int age;
    float height;
    char name[50];

    printf("Enter your name: ");
    scanf("%49s", name);      /* %49s reads at most 49 chars — prevents overflow */

    printf("Enter your age: ");
    scanf("%d", &age);        /* & = address-of: scanf needs to WRITE to age */

    printf("Enter height (m): ");
    scanf("%f", &height);

    printf("%s is %d years old and %.2fm tall.\n", name, age, height);
    return 0;
}

Key point: scanf takes pointers to where it should store the values. That's why you write &age — you're passing the address of age, not its value. For strings (char[]), the array name already decays to a pointer, so no & is needed (but the width limit is important).

Diff: Wrong Format Specifier

/* BROKEN: %d used for a float */
#include <stdio.h>

int main(void) {
    float temp = 98.6;
    printf("Temperature: %d\n", temp);
    return 0;
}
/* FIXED: %f for float */
#include <stdio.h>

int main(void) {
    float temp = 98.6f;
    printf("Temperature: %.1f\n", temp);
    return 0;
}
What was wrong: %d expects an int argument. Passing a float where %d is expected is undefined behaviour — you'll typically see garbage output like Temperature: -1073741824. GCC with -Wall will warn: "format '%d' expects argument of type 'int'".

Why the fix works: %f is the correct specifier for float and double. The %.1f variant means "one decimal place". Adding the f suffix to 98.6f makes it a float literal (without it, 98.6 is a double).
⚠ Common Mistakes
  • Forgetting & in scanf — scanf("%d", age) passes the value of age as an address, which is almost certainly invalid. Always use scanf("%d", &age).
  • Integer overflow — int maxes out at ~2.1 billion. Computing 1000000 * 1000000 as int silently overflows to a wrong value. Use long long or check your range.
  • char sign — Whether char is signed or unsigned is implementation-defined. For clarity, explicitly write signed char or unsigned char when it matters.

Practice Exercises

Basic Type size explorer

Write a program that prints the size in bytes of every fundamental type using sizeof. Include: char, short, int, long, long long, float, double, long double.

Show hint
Use printf("%zu\n", sizeof(int)). The %zu specifier is for size_t (the type returned by sizeof).
Intermediate Simple calculator

Write a program that reads two integers and an operator character (+, -, *, /) from the user, then prints the result. Handle division by zero.

Show hint
Use scanf("%d %c %d", &a, &op, &b). Use an if/else chain on op to select the operation. Check b == 0 before division.
Challenge Bit manipulation

Write a function that uses bitwise operators (&, |, ^, ~, <<, >>) to: (1) set a specific bit in an integer, (2) clear a specific bit, (3) toggle a specific bit, (4) check if a specific bit is set. Test with various inputs.

Show hint
Set bit n: x |= (1 << n). Clear bit n: x &= ~(1 << n). Toggle: x ^= (1 << n). Check: (x >> n) & 1.

Checkpoint Quiz

Test your understanding

1. Which format specifier prints a float with printf?
%f is used for float and double in printf. %d and %i are for integers. Mixing them is undefined behaviour.
2. What is the result of 7 % 3 in C?
% is the modulo operator. 7 = 2×3 + 1, so the remainder is 1.
3. Which declaration creates a constant integer?
const int x = 5; declares a constant. static changes the storage duration (survives function calls) but doesn't prevent reassignment.
