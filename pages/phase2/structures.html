Structures

Why does this matter?

Structs are the foundation of C-based databases and data formats. SQLite's entire internal representation — B-tree pages, cells, cursors — is built from structs. The Linux kernel's task_struct (representing a process) has hundreds of fields. Network protocols like TCP/IP are specified as struct layouts that map directly to byte-level wire formats. Structs are how C represents the real world.

Defining and Using Structs

#include <stdio.h>
#include <string.h>

/* Define a struct type */
struct Point {
    double x;
    double y;
};

struct Student {
    char name[50];
    int  age;
    double gpa;
};

int main(void) {
    /* Declare and initialise */
    struct Point p1 = {3.0, 4.0};
    struct Student s1 = {"Alice", 20, 3.85};

    /* Access fields with . operator */
    printf("(%g, %g)\n", p1.x, p1.y);
    printf("%s, age %d, GPA %.2f\n", s1.name, s1.age, s1.gpa);

    /* Modify a field */
    s1.age = 21;
    strncpy(s1.name, "Alicia", sizeof(s1.name) - 1);

    return 0;
}

typedef for Cleaner Syntax

typedef creates a type alias so you don't need to write struct every time:

/* Method 1: separate typedef */
struct Point { double x; double y; };
typedef struct Point Point;

/* Method 2: combined (most common) */
typedef struct {
    double x;
    double y;
} Point;

/* Method 3: self-referential structs (linked lists need the tag) */
typedef struct Node {
    int value;
    struct Node *next;  /* must use "struct Node" here, not just "Node" */
} Node;

/* Usage */
Point p = {1.0, 2.0};  /* no "struct" keyword needed */

Structs and Pointers

When you have a pointer to a struct, use the -> operator (arrow) to access fields. It's shorthand for (*ptr).field:

#include <stdio.h>
#include <stdlib.h>
#include <string.h>

typedef struct {
    char name[50];
    int age;
} Person;

void birthday(Person *p) {
    p->age++;   /* equivalent to: (*p).age++ */
}

int main(void) {
    Person alice = {"Alice", 30};
    birthday(&alice);
    printf("%s is now %d\n", alice.name, alice.age);  /* Alice is now 31 */

    /* Heap-allocated struct */
    Person *bob = malloc(sizeof(Person));
    if (!bob) return 1;
    strncpy(bob->name, "Bob", sizeof(bob->name) - 1);
    bob->age = 25;

    printf("%s is %d\n", bob->name, bob->age);

    free(bob);
    return 0;
}

Arrays of Structs

#include <stdio.h>
#include <string.h>

typedef struct { char name[30]; float score; } Student;

int main(void) {
    Student class[] = {
        {"Alice", 92.5},
        {"Bob",   78.0},
        {"Carol", 88.5},
    };
    int n = sizeof(class) / sizeof(class[0]);

    /* Find highest scorer */
    int best = 0;
    for (int i = 1; i < n; i++) {
        if (class[i].score > class[best].score) best = i;
    }
    printf("Top student: %s (%.1f)\n", class[best].name, class[best].score);
    return 0;
}

Diff: Using . Instead of -> on a Pointer

/* BROKEN: dot operator on a pointer */
#include <stdio.h>
#include <stdlib.h>

typedef struct { int x; int y; } Point;

int main(void) {
    Point *p = malloc(sizeof(Point));
    p.x = 10;   /* ERROR: p is a pointer, not a struct */
    p.y = 20;
    printf("%d %d\n", p.x, p.y);
    free(p);
    return 0;
}
/* FIXED: arrow operator for struct pointers */
#include <stdio.h>
#include <stdlib.h>

typedef struct { int x; int y; } Point;

int main(void) {
    Point *p = malloc(sizeof(Point));
    if (!p) return 1;
    p->x = 10;   /* FIXED: arrow for pointer access */
    p->y = 20;
    printf("%d %d\n", p->x, p->y);
    free(p);
    return 0;
}
What was wrong: p.x uses the dot operator, which works when p is a struct variable. But p is a Point * (pointer), so the compiler produces: "error: request for member 'x' in something not a structure or union".

Why the fix works: p->x is the arrow operator — it dereferences the pointer and accesses the field in one step. It's exactly equivalent to (*p).x, just more readable.
⚠ Common Mistakes
  • Copying structs accidentally — Passing a large struct by value copies all its bytes. For structs with string buffers, this can be expensive. Pass large structs by pointer when performance matters.
  • struct padding and sizeof — sizeof(struct) can be larger than the sum of its fields due to alignment padding. Never assume the struct size.
  • Self-referential typedef without the tag — A linked list node needs struct Node *next inside the struct definition. The typedef name Node isn't available until the definition is complete, so you must use the struct tag inside.

Practice Exercises

Basic Rectangle area and perimeter

Define a struct Rectangle with double width and double height. Write functions double area(Rectangle r) and double perimeter(Rectangle r). Test with several rectangles.

Show hint
area = width * height; perimeter = 2 * (width + height). Pass the struct by value since it's small — two doubles.
Intermediate Student grade tracker

Create an array of 5 struct Student values with name and grade. Write functions to find the average grade, the highest scorer, and to sort the array by grade (descending).

Show hint
Use qsort() with a comparator function, or implement bubble sort manually. The comparator for descending order: return (int)(b.score - a.score).
Challenge Implement a date struct with comparison

Define struct Date { int year; int month; int day; }. Write: (1) a function to check if a date is valid, (2) a comparison function (returns -1/0/1), (3) a function that returns the number of days between two dates. Test thoroughly.

Show hint
For days between dates: convert each date to a "Julian Day Number" (integer count from a reference date), then subtract. Look up the algorithm for Julian Day Number — it handles month lengths and leap years cleanly.

Checkpoint Quiz

Test your understanding

1. How do you access a field of a struct through a pointer?
p->field is the arrow operator for struct pointers. It's shorthand for (*p).field. Use the dot (.) when you have the struct itself, arrow (->) when you have a pointer to the struct.
2. What does typedef do when used with a struct?
typedef creates a type alias. Without it, you must write "struct Point" everywhere. With typedef, you can write just "Point".
3. Structs are passed to functions by:
C passes structs by value — the whole struct is copied. For large structs or when you need to modify the original, explicitly pass a pointer.
