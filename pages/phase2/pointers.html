Pointers

Why does this matter?

Pointers are what gave C its power to write operating system kernels. The Linux kernel uses pointers to manage virtual memory, device drivers use pointers to access hardware registers at specific addresses, and every call to malloc() returns a pointer. Pointers also enable data structures like linked lists and trees. Avoiding pointers means staying in C's shallow end forever — mastering them opens the entire language.

What is a Pointer?

A pointer is a variable that holds a memory address. Think of memory as a row of numbered boxes — a pointer is a note that says "the answer is in box #4021."

#include <stdio.h>

int main(void) {
    int x = 42;
    int *p = &x;   /* p holds the address of x */
                   /* * in declaration = "p is a pointer to int" */
                   /* & = "address of" operator */

    printf("Value of x:       %d\n",  x);    /* 42 */
    printf("Address of x:     %p\n",  &x);   /* e.g. 0x7fff5abc1234 */
    printf("Value of p:       %p\n",  p);    /* same address */
    printf("Value pointed to: %d\n",  *p);   /* 42 — dereference */

    *p = 100;        /* change x through the pointer */
    printf("x is now: %d\n", x);   /* 100 */

    return 0;
}

Pointer Arithmetic

When you add an integer to a pointer, C automatically scales by the size of the pointed-to type. This is how arrays work internally:

#include <stdio.h>

int main(void) {
    int arr[5] = {10, 20, 30, 40, 50};
    int *p = arr;   /* p points to arr[0] — array name decays to pointer */

    printf("%d\n", *p);       /* 10 — arr[0] */
    printf("%d\n", *(p+1));   /* 20 — arr[1] */
    printf("%d\n", *(p+2));   /* 30 — arr[2] */

    p++;             /* advance pointer by 1 int (4 bytes) */
    printf("%d\n", *p);   /* 20 */

    /* These are equivalent */
    printf("%d == %d\n", arr[3], *(arr + 3));  /* 40 == 40 */

    return 0;
}

Pointers and Functions: Pass by Reference

By passing a pointer, you give a function the ability to modify the original variable:

#include <stdio.h>

void swap(int *a, int *b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}

void increment(int *n) {
    (*n)++;   /* parentheses needed: *n++ would increment the pointer! */
}

int main(void) {
    int x = 5, y = 10;
    printf("Before: x=%d y=%d\n", x, y);
    swap(&x, &y);
    printf("After:  x=%d y=%d\n", x, y);  /* x=10 y=5 */

    increment(&x);
    printf("x after increment: %d\n", x);  /* 11 */
    return 0;
}

Dynamic Memory with malloc and free

malloc() allocates memory on the heap. Unlike stack variables, heap memory persists until you explicitly free() it:

#include <stdio.h>
#include <stdlib.h>

int main(void) {
    int n = 5;

    /* Allocate array of n ints on the heap */
    int *arr = malloc(n * sizeof(int));
    if (arr == NULL) {   /* ALWAYS check for NULL — malloc can fail */
        fprintf(stderr, "malloc failed\n");
        return 1;
    }

    /* Use it like a regular array */
    for (int i = 0; i < n; i++) {
        arr[i] = i * 10;
    }

    for (int i = 0; i < n; i++) {
        printf("%d ", arr[i]);  /* 0 10 20 30 40 */
    }
    printf("\n");

    free(arr);   /* ALWAYS free when done — prevents memory leaks */
    arr = NULL;  /* good habit: NULL the pointer after freeing */

    return 0;
}

Null Pointer

A null pointer is a pointer with value NULL (typically 0). It points to nothing. Dereferencing it causes a segfault:

int *p = NULL;
*p = 5;    /* SEGFAULT — cannot dereference NULL */

/* Always check before dereferencing */
if (p != NULL) {
    *p = 5;
}

Diff: Forgetting to free() — Memory Leak

/* BROKEN: memory leak */
#include <stdlib.h>

void process(int n) {
    int *data = malloc(n * sizeof(int));
    if (!data) return;

    /* ... use data ... */

    /* BUG: forgot to free! */
}   /* data goes out of scope but heap memory is NOT freed */

int main(void) {
    for (int i = 0; i < 1000; i++) {
        process(1000);   /* leaks 4000 bytes each call */
    }   /* total leak: ~4 MB */
    return 0;
}
/* FIXED: always free allocated memory */
#include <stdlib.h>

void process(int n) {
    int *data = malloc(n * sizeof(int));
    if (!data) return;

    /* ... use data ... */

    free(data);   /* FIXED: release heap memory before return */
}

int main(void) {
    for (int i = 0; i < 1000; i++) {
        process(1000);   /* no leak */
    }
    return 0;
}
What was wrong: Every call to malloc() that isn't matched by a free() is a memory leak. The heap memory is never returned to the OS while the process runs. In a loop, this accumulates rapidly. Long-running servers with memory leaks eventually crash or are killed by the OOM killer.

Why the fix works: free(data) returns the heap block to the allocator. Every code path in the function that allocates memory must have a corresponding free() on every exit path (including early returns on error).
⚠ Common Mistakes
  • Dereferencing an uninitialised pointer — int *p; *p = 5; is undefined behaviour. p contains garbage — an arbitrary address. Always initialise pointers.
  • Use after free — Reading or writing through a pointer after calling free() is undefined behaviour. Set pointers to NULL after freeing them.
  • Double free — Calling free() twice on the same pointer corrupts the heap allocator's metadata. This is a common security vulnerability.

Practice Exercises

Basic Swap without a temp variable

Write a function void swap_xor(int *a, int *b) that swaps two integers using XOR without a temporary variable. Verify it works with various inputs including negative numbers.

Show hint
XOR swap: *a ^= *b; *b ^= *a; *a ^= *b; — but be careful: if a and b point to the same location, this will zero it out! The standard temp-variable swap is safer in practice.
Intermediate Dynamic array with resize

Use malloc and realloc to build a dynamically-growing integer array. Start with capacity 4 and double it each time the array is full. Push 20 numbers and then print them all. Free at the end.

Show hint
Track both size (current count) and capacity. When size == capacity, call realloc(ptr, new_capacity * sizeof(int)) and check the return value for NULL before updating the pointer.
Challenge Implement qsort comparator for structs

Create an array of struct Person { char name[30]; int age; }. Use the standard library qsort() function with a custom comparator to sort by age (ascending), then by name (alphabetical). Print the sorted results.

Show hint
qsort comparator signature: int cmp(const void *a, const void *b). Cast: const struct Person *pa = a; Then compare pa->age vs pb->age. For name sort: use strcmp(pa->name, pb->name).

Checkpoint Quiz

Test your understanding

1. What does the & operator do when applied to a variable?
& is the address-of operator. &x gives the memory address where x is stored. (& in a declaration means reference in C++ but not in C.)
2. What does *ptr do when ptr is a pointer?
The * dereference operator reads (or writes) the value stored at the address held by ptr. * in a declaration means "pointer to"; * in an expression means "value at".
3. Which function allocates memory on the heap?
malloc(n) allocates n bytes on the heap and returns a void* pointer to the memory. new is a C++ keyword. alloc and halloc don't exist in standard C.
