Recursion

Why does this matter?

Compilers use recursive descent parsers to read source code. File systems use recursive tree traversal to list directories. Quicksort and mergesort are inherently recursive. When you call ls -R, the underlying C code is recursing through directory inodes. Understanding recursion — especially when not to use it — is a prerequisite for reading and writing production systems code.

What is Recursion?

Recursion is when a function calls itself. Every recursive solution has two parts:

  • Base case — the condition that stops recursion (returns without calling itself)
  • Recursive case — reduces the problem toward the base case and calls itself
#include <stdio.h>

/* factorial: n! = n * (n-1) * (n-2) * ... * 1 */
long factorial(int n) {
    if (n <= 1) return 1;       /* base case */
    return n * factorial(n - 1); /* recursive case: reduces n by 1 */
}

int main(void) {
    for (int i = 0; i <= 10; i++) {
        printf("%d! = %ld\n", i, factorial(i));
    }
    return 0;
}

Fibonacci — Naive vs Memoised

#include <stdio.h>
#include <string.h>

/* Naive recursive Fibonacci — O(2^n) time */
long fib_naive(int n) {
    if (n <= 1) return n;
    return fib_naive(n - 1) + fib_naive(n - 2);
}

/* Memoised Fibonacci — O(n) time, O(n) space */
#define MAXN 100
long memo[MAXN];

long fib_memo(int n) {
    if (n <= 1) return n;
    if (memo[n] != -1) return memo[n];       /* cached result */
    memo[n] = fib_memo(n-1) + fib_memo(n-2);
    return memo[n];
}

/* Iterative Fibonacci — O(n) time, O(1) space */
long fib_iter(int n) {
    if (n <= 1) return n;
    long a = 0, b = 1;
    for (int i = 2; i <= n; i++) {
        long tmp = a + b;
        a = b;
        b = tmp;
    }
    return b;
}

int main(void) {
    memset(memo, -1, sizeof(memo));

    /* Compare all three — they produce the same results */
    for (int i = 0; i <= 10; i++) {
        printf("fib(%2d) = %ld\n", i, fib_iter(i));
    }
    return 0;
}

Practical Recursion: Directory Tree Walk

#include <stdio.h>
#include <dirent.h>
#include <string.h>

void walk_dir(const char *path, int depth) {
    DIR *dir = opendir(path);
    if (!dir) return;

    struct dirent *entry;
    while ((entry = readdir(dir)) != NULL) {
        if (strcmp(entry->d_name, ".") == 0 || strcmp(entry->d_name, "..") == 0) {
            continue;
        }
        /* Indent by depth */
        for (int i = 0; i < depth; i++) printf("  ");
        printf("%s\n", entry->d_name);

        if (entry->d_type == DT_DIR) {
            char subpath[1024];
            snprintf(subpath, sizeof(subpath), "%s/%s", path, entry->d_name);
            walk_dir(subpath, depth + 1);  /* recurse into subdirectory */
        }
    }
    closedir(dir);
}

int main(void) {
    walk_dir(".", 0);
    return 0;
}
⚠ Common Mistakes
  • No base case — Infinite recursion eventually hits the stack size limit and crashes with a segfault (stack overflow). Always verify your base case terminates for all valid inputs.
  • Base case unreachable — If the recursive call doesn't move toward the base case (e.g., calling f(n) instead of f(n-1)), you still get infinite recursion.
  • Exponential time without memoisation — Naive recursive Fibonacci computes fib(n-1) and fib(n-2) independently, redoing vast amounts of work. For fib(40), this is ~2 billion function calls. Always memoize or convert to iteration for repeated subproblems.

Practice Exercises

Basic Sum of digits

Write a recursive function int digit_sum(int n) that returns the sum of all digits of a non-negative integer. E.g., digit_sum(123) → 6.

Show hint
Base case: n < 10, return n. Recursive case: return (n % 10) + digit_sum(n / 10). Integer division drops the last digit.
Intermediate Binary search (recursive)

Implement recursive binary search: int bsearch_r(int *arr, int lo, int hi, int target). Returns the index of target or -1 if not found. The array is sorted in ascending order.

Show hint
Base case: lo > hi → return -1. Mid = (lo+hi)/2. If arr[mid]==target return mid. If target < arr[mid], recurse on left half (lo, mid-1). Otherwise recurse on right (mid+1, hi).
Challenge Implement merge sort

Implement merge sort recursively. It should sort an integer array in place (using a temporary buffer for the merge step). Verify it runs in O(n log n) by timing it against a large array.

Show hint
Split array in half, recursively sort each half, then merge the two sorted halves into a temp buffer, and copy back. The merge step is O(n); there are O(log n) levels of recursion. Use clock() from time.h to measure.

Checkpoint Quiz

Test your understanding

1. What is the base case in recursion?
The base case returns a value without making another recursive call. Without it, recursion never terminates.
2. What happens if a recursive function has no base case?
Without a base case the function calls itself forever, pushing frames until the stack exhausts its ~8 MB limit, causing SIGSEGV.
3. Compared to an iterative solution, recursion typically:
Each recursive call pushes a stack frame (local variables, return address, saved registers). Deep recursion can overflow the stack.
